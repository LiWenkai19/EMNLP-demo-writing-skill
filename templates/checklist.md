# 投稿前自检清单

提交前逐条打勾。任何「❌ 硬性」项未过 → 大概率 desk reject。

## A. 硬性门槛（缺一直接桌拒）

- [ ] **正文 ≤ 6 页**（录用后才 +1 页；参考文献/伦理声明不限，附录 ≤ 2 页）
- [ ] **报告了某种 evaluation**（可用性问卷 / 人工评测 / 效率指标，任一即可）
- [ ] **正文给出可运行入口**：在线 demo 网址 **或** 可下载安装包链接
- [ ] **演示视频 ≤ 2.5 分钟**，链接可公开访问或作为补充材料提交
- [ ] 符合 EMNLP 2026 官方 LaTeX 模板格式，PDF 提交
- [ ] **单盲**：作者信息可见（demo track 不匿名），不要误用匿名模板

## B. 官方 8 个核心问题（评委逐条打分，确保都有着落）

- [ ] 1. 解决什么问题？（Introduction 第 1–2 段）
- [ ] 2. 为什么重要、影响何在？（Introduction 痛点+影响）
- [ ] 3. 方法/技术的新颖性？（Introduction 贡献点 + System 节）
- [ ] 4. 目标用户是谁？（Introduction / Usage 节明确点名）
- [ ] 5. 系统如何工作？（System / Architecture 节）
- [ ] 6. 与现有系统的比较？（Related Work + Evaluation）
- [ ] 7. 系统的 license？（脚注或 Conclusion，写明具体协议）
- [ ] 8. 如何评估、有无用户研究？（Evaluation 节）

## C. 结构完整性

- [ ] 有 6 段骨架：Intro / Related Work / System / Usage / Evaluation / Conclusion
- [ ] System + Usage 占最大篇幅（重心在系统与使用）
- [ ] 有一张系统总览图 Figure 1
- [ ] Usage 节有界面截图
- [ ] 引言遵循「背景 → However 痛点 → We present X → 贡献点」四步

## D. 风格与链接

- [ ] 系统命名为「Name: 一句话定位」
- [ ] 摘要含 demo/视频/代码链接
- [ ] 正文给齐三个链接：GitHub、在线 demo、演示视频
- [ ] 明确写出开源 license（如 Apache-2.0 / MIT）
- [ ] 有 Limitations（诚实写局限，不夸大）
- [ ] 时态正确（背景现在完成时 / 系统一般现在时 / 实验一般过去时）

## E. 投稿事务

- [ ] 至少一名作者能到布达佩斯现场 live demo（2026/10/24–29）
- [ ] 注意截止：投稿 2026/7/10，通知 2026/8/20，camera-ready 2026/8/30
- [ ] 无并行投稿冲突，同作者多投不超过 25% 内容重叠
- [ ] 伦理声明符合 ACM Code of Ethics
